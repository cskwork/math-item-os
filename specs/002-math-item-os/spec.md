# Feature Specification: Math Knowledge Graph + Item OS

**Feature Branch**: `002-math-item-os`
**Created**: 2026-04-07
**Status**: Draft
**Input**: User description: "Math Knowledge Graph + Item OS - LaTeX 수식 기반 수학 문항 저장/분류/검색/추천/생성 교사용 플랫폼"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 수학 문항 등록 및 수식 렌더링 (Priority: P1)

교사가 LaTeX 수식이 포함된 수학 문항을 시스템에 등록하면,
수식이 브라우저에서 정확하게 렌더링되고 검증 가능한 형태로 저장된다.
문항은 LaTeX(저작) + MathML(렌더링) + CAS 수식 트리(검증/비교)의
3중 표현으로 자동 변환되어 저장된다.

**Why this priority**: 모든 후속 기능(검색, 유사도, 생성)의 기반이 되는
데이터 입력 경로. 수식이 정확히 저장/렌더링되지 않으면 플랫폼 전체가 무의미.

**Independent Test**: LaTeX 수식이 포함된 문항 10건을 등록하고,
각 문항의 수식이 브라우저에서 올바르게 표시되며 CAS 트리로 변환되었는지 확인.

**Acceptance Scenarios**:

1. **Given** 교사가 LaTeX 수식 `2(x-3)=10`이 포함된 문항을 입력,
   **When** 저장 버튼을 클릭,
   **Then** 문항이 저장되고 수식이 브라우저에서 렌더링되며,
   CAS 수식 트리가 자동 생성되어 수식 동치 검증이 가능한 상태가 된다.

2. **Given** CSV/JSON/QTI 형식의 문항 파일(1만건 이하),
   **When** 일괄 업로드,
   **Then** 각 문항의 LaTeX 수식이 파싱되어 3중 표현으로 변환되고,
   파싱 실패 건은 오류 리포트와 함께 별도 표시된다.

3. **Given** 이미 등록된 문항,
   **When** 교사가 수식이나 보기를 수정,
   **Then** 새 버전으로 저장되고 이전 버전 이력이 보존된다.

### User Story 2 - 교육과정 기반 문항 분류 및 검색 (Priority: P1)

교사가 학제/학년/단원/스킬/난이도/문항유형 등 메타데이터로 문항을 분류하고,
한국어 자연어 + 구조 필터로 빠르게 검색하여 원하는 문항을 찾는다.

**Why this priority**: 문항 등록 직후 가장 먼저 필요한 기능.
8만+ 문항에서 목적에 맞는 문항을 빠르게 찾지 못하면 플랫폼 가치가 없다.

**Independent Test**: 등록된 문항 1,000건 이상에서 "중2 일차방정식 분배법칙
중급 난이도"로 검색하여 관련 문항이 1.5초 이내에 반환되는지 확인.

**Acceptance Scenarios**:

1. **Given** 문항 등록 시,
   **When** 교사가 학제(초/중/고), 학년/학기, 과목, 단원, 스킬, 성취기준,
   난이도, 문항유형, 사용목적(진단/보충/시험직전/심화 등)을 태깅,
   **Then** 해당 메타데이터가 문항에 연결되고 검색 인덱스에 반영된다.

2. **Given** 8만건 이상의 문항이 등록된 상태,
   **When** 교사가 "분배법칙 일차방정식"으로 검색하고
   학년=중2, 난이도=중급 필터를 적용,
   **Then** 관련 문항 목록이 1.5초 이내에 표시되고
   수식이 렌더링된 상태로 미리보기 제공.

3. **Given** 검색 결과 목록,
   **When** 교사가 특정 문항을 선택,
   **Then** 문항 본문, 수식, 풀이, 메타데이터(스킬/성취기준/난이도),
   연결된 선수학습 스킬이 한 화면에 표시된다.

### User Story 3 - 선수학습 그래프 탐색 (Priority: P2)

교사가 특정 스킬의 선수학습 관계를 시각적 그래프로 탐색하여,
학생이 어디에서 막혔는지 진단하고 복습 경로를 파악한다.

**Why this priority**: "왜 이 학생이 이 문제를 못 푸는지" 진단하려면
선수학습 의존 관계를 볼 수 있어야 한다. 단순 문제은행과의 핵심 차별점.

**Independent Test**: "일차방정식 풀기" 스킬을 선택하면 선수 스킬
(분배법칙, 이항, 등식 성질 등)이 방향성 그래프로 표시되고,
각 노드를 클릭하면 해당 스킬의 문항 목록으로 이동하는지 확인.

**Acceptance Scenarios**:

1. **Given** 스킬 간 선수학습 관계(강한/약한)가 정의된 상태,
   **When** 교사가 특정 스킬 노드를 선택,
   **Then** 해당 스킬의 선수학습 경로가 방향성 비순환 그래프(DAG)로
   시각화되고, 각 노드에 해당 스킬의 문항 수와 난이도 분포가 표시된다.

2. **Given** 그래프에서 특정 선수 스킬 노드를 클릭,
   **When** "이 스킬 문항 보기" 선택,
   **Then** 해당 스킬에 연결된 문항 목록이 난이도순으로 표시되어
   복습용 문항을 바로 선택할 수 있다.

### User Story 4 - 구조적 유사문항 검색 (Priority: P2)

교사가 특정 문항의 "구조적으로 유사한" 문항을 검색하면,
단순 텍스트 유사도가 아닌 수식 구조/스킬/난이도/오개념 기반으로
유사 문항이 랭킹되어 표시되고, 각 결과에 "왜 유사한지" 설명이 제공된다.

**Why this priority**: 기존 플랫폼(매쓰플랫 등)의 "숫자만 바꾼" 유사문제와
차별화되는 핵심 기능. 수식 구조 + 스킬 + 오개념 결합 검색은 시장에 없다.

**Independent Test**: `2(x-3)=10` 문항에 대해 유사문항 검색 시,
단순 숫자 변형이 아닌 구조적으로 유사한 문항(예: `3(x+2)=15`,
분배법칙이 필요한 다른 일차방정식)이 상위에 랭킹되고,
각 결과에 유사 근거("같은 스킬, 유사한 수식 구조, 난이도 근접")가
표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 등록된 문항 중 하나를 선택,
   **When** "유사문항 찾기" 실행,
   **Then** 스킬 일치도, 수식 구조 유사도, 선수학습 거리, 난이도 근접도,
   오개념 프로파일을 결합한 랭킹으로 상위 20개 유사 문항이 표시되고,
   각 결과에 유사 근거 설명이 포함된다.

2. **Given** 유사문항 검색 결과,
   **When** 교사가 특정 결과에 "관련 없음" 피드백,
   **Then** 해당 피드백이 기록되어 이후 랭킹에 반영된다.

---

### User Story 5 - 오개념 기반 교정 문항 추천 (Priority: P3)

교사가 학생의 오류 패턴(부호 이동 오류, 분배법칙 누락 등)을 선택하면,
해당 오개념을 교정하기 위한 문항이 추천되고 교정 경로가 제안된다.

**Why this priority**: ALEKS만 부분 지원하며 한국 교육과정에 맞는
오개념 교정 추천은 시장에 없다. 교사의 개별화 지도를 직접 지원.

**Independent Test**: "부호 이동 오류(sign_error_transposition)" 오개념을
선택하면, 이 오류를 드러내는 문항 + 교정용 단계별 문항이 추천되고,
추천 근거("부호 이동 오류를 교정하는 패턴 포함")가 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 오개념 태그가 연결된 문항들이 존재,
   **When** 교사가 특정 오개념을 선택하고 "교정 문항 추천" 실행,
   **Then** 해당 오개념을 진단하는 문항 + 교정 경로(선수 스킬 복습 ->
   기초 연습 -> 확인 문제)가 단계별로 제안되고,
   각 추천에 "왜 이 문제인지" 설명이 포함된다.

2. **Given** 추천된 교정 문항 세트,
   **When** 교사가 난이도나 문항 수를 조정,
   **Then** 같은 교정 경로 내에서 대체 문항으로 교체되어 제공된다.

---

### User Story 6 - 템플릿 기반 유사문항 자동 생성 (Priority: P3)

교사가 기존 문항을 기반으로 구조적 변형 문항을 자동 생성하면,
숫자만 바꾸는 것이 아닌 풀이 단계/계수 범위/표현형을 제어한
변형 문항이 생성되고, 정답/동치성/유일해가 자동 검증된다.

**Why this priority**: 학습지 제작 시 "같은 유형 다른 문제" 수요가 크나,
기존 플랫폼은 숫자 변형에 그침. 구조적 생성 + 자동 검증은 시장 부재.

**Independent Test**: `2(x-3)=10` 문항에서 "난이도 유지, 분수 계수 포함,
서술형 변환" 옵션으로 변형 5건을 생성하고, 각 생성물의 정답이 자동
검증되며 풀이가 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 기존 문항을 원본으로 선택,
   **When** 교사가 제어 변수(풀이 단계 수, 계수 범위, 분수/음수 포함,
   표현형) 설정 후 "변형 문항 생성" 실행,
   **Then** 설정에 맞는 변형 문항이 생성되고, 각 문항의 정답 동치성과
   유일해가 CAS 기반으로 자동 검증되어 결과가 표시된다.

2. **Given** 자동 생성된 변형 문항,
   **When** CAS 자동 검증이 실패,
   **Then** 해당 문항은 배포 금지 상태로 표시되고
   실패 사유가 교사에게 안내된다.

3. **Given** 검증 통과한 생성 문항,
   **When** 교사가 승인,
   **Then** 원본 문항과의 관계가 추적 가능하게 저장되고,
   "AI 생성" 라벨이 자동 부착된다.

---

### User Story 7 - 목적별 학습지 제작 (Priority: P4)

교사가 진단/보충/시험직전/심화 등 목적을 선택하고 학생 수준을 지정하면,
시스템이 추천 근거와 함께 문항 세트를 제안하고,
교사가 조정하여 학습지를 완성한다.

**Why this priority**: 북극성 지표("목적에 맞는 문항 세트 제작 시간 단축")를
직접 달성하는 최종 사용자 워크플로우.

**Independent Test**: "중2 일차방정식 분배법칙, 보충용, 쉬운 난이도 5문항
+ 유사문항 각 3개"를 30초 이내에 완성하고 PDF/온라인 학습지로 출력 확인.

**Acceptance Scenarios**:

1. **Given** 교사가 목적(보충), 단원(일차방정식), 난이도(쉬움),
   문항 수(5), 유사문항 수(각 3)를 설정,
   **When** "학습지 생성" 실행,
   **Then** 조건에 맞는 문항 세트가 추천 근거와 함께 제안되고,
   전체 소요 시간이 30초 이내.

2. **Given** 제안된 문항 세트,
   **When** 교사가 특정 문항을 교체하거나 순서를 변경,
   **Then** 변경이 즉시 반영되고 PDF 및 온라인 형태로 출력 가능.

---

### Edge Cases

- LaTeX 수식 파싱 실패 시 원본 LaTeX 텍스트를 보존하고 오류 표시
- 선수학습 그래프에 순환(cycle)이 감지되면 경고 및 해당 엣지 비활성화
- 동일 문항 중복 등록 시 수식 구조 기반 중복 탐지 및 경고
- 생성 문항의 CAS 검증 타임아웃(10초) 시 "검증 미완료" 상태로 표시
- 오개념 태그가 없는 문항에 대한 교정 추천 시 "태그 부족" 안내
- 대량 업로드(1만건) 중 네트워크 중단 시 진행상황 보존 및 재시도

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 LaTeX 수식이 포함된 문항을 등록하고 LaTeX + MathML +
  CAS 수식 트리 3중 표현으로 자동 변환하여 저장해야 한다(MUST).
- **FR-002**: 시스템은 CSV, JSON, QTI 3 형식의 일괄 업로드를 지원하고
  1회당 1만건까지 처리해야 한다(MUST).
- **FR-003**: 시스템은 문항의 수정 시 버전 이력을 보존해야 한다(MUST).
- **FR-004**: 시스템은 한국 2022 개정 교육과정 기반 메타데이터
  (학제/학년/단원/스킬/성취기준/난이도/문항유형/사용목적)로
  문항을 분류해야 한다(MUST).
- **FR-005**: 시스템은 한국어 자연어 검색 + 구조 필터 조합으로
  8만건 이상에서 1.5초 이내 검색 결과를 반환해야 한다(MUST).
- **FR-006**: 시스템은 스킬 간 선수학습 관계(강한/약한)를
  방향성 비순환 그래프로 저장하고 시각적으로 탐색 가능해야 한다(MUST).
- **FR-007**: 시스템은 스킬 일치도 + 수식 구조 유사도 + 선수학습 거리 +
  텍스트 의미 유사도 + 난이도 근접도 + 오개념 프로파일을 결합하여
  유사문항을 랭킹하고, 각 결과에 유사 근거를 제공해야 한다(MUST).
- **FR-008**: 시스템은 오개념별 교정 문항 추천과 단계별 교정 경로를
  설명 가능한 형태로 제안해야 한다(MUST).
- **FR-009**: 시스템은 기존 문항 기반으로 제어 변수(풀이 단계, 계수 범위,
  분수/음수, 표현형)를 조정한 변형 문항을 생성하고 CAS 기반 자동 검증
  (정답 동치성, 유일해)을 수행해야 한다(MUST).
- **FR-010**: CAS 자동 검증에 실패한 생성 문항은 배포가 금지되어야 한다(MUST).
- **FR-011**: AI로 생성된 문항에는 "AI 생성" 라벨이 자동 부착되어야 한다(MUST).
- **FR-012**: 시스템은 모든 추천에 설명 가능한 근거를 포함해야 한다(MUST).
- **FR-013**: 시스템은 목적별(진단/보충/시험직전/심화 등) 학습지를
  문항 추천 + 교사 조정 + PDF/공유 링크 출력으로 제작할 수 있어야 한다(MUST).
  학생 직접 접근/응답 수집은 범위 밖이며, 학습지는 외부 배포용으로만 생성한다.
- **FR-014**: 시스템은 3단계 역할 기반 접근 제어(RBAC)를 제공해야 한다(MUST).
  역할: 시스템 관리자, 콘텐츠 검수자, 교사(일반).
  MVP에서는 시스템 관리자 + 콘텐츠 검수자가 주 사용자이며,
  교사(일반) 역할 오픈은 이후 단계로 진행한다.
  문항 품질 상태(draft/reviewed/approved/retired)를 관리해야 한다(MUST).
  상태 전이 규칙: 순방향 draft→reviewed→approved→retired,
  역전이 approved→draft 허용(수정 필요 시).
  검수자: draft↔reviewed↔approved 전이 가능.
  관리자: retired 전이 및 모든 역전이 승인 가능.
- **FR-015**: 시스템은 감사 로그를 영구 보존하고 문항 변경/생성/추천
  이벤트를 추적해야 한다(MUST).

### Key Entities

- **Item**: 문항 본체. LaTeX + MathML + CAS 수식 트리 3중 표현,
  메타태그(학제/학년/단원/스킬/난이도/유형/목적/품질상태)
- **Skill**: 수학 스킬 노드 (예: solve_linear_eq_distributive).
  스킬 간 선수학습 관계(강한/약한)로 DAG 형성
- **Standard**: 2022 개정 교육과정 성취기준 (예: KR2022-M2-ALG-1)
- **Misconception**: 오개념 (예: sign_error_transposition).
  수학교육 연구 기반 사전 정의 목록을 초기 데이터로 제공하며,
  콘텐츠 검수자가 추가/수정 가능. 문항-오개념 연결 및 교정 경로
- **DifficultyProfile**: 난이도 프로파일. MVP에서는 5단계 정수
  (1=매우 쉬움 ~ 5=매우 어려움), 검수자 수동 설정.
  향후 행동 데이터 축적 시 IRT 파라미터 기반 보정 추가
- **Solution**: 단계형 풀이 + 해설
- **Template**: 파라미터화된 문제 구조 (생성용)
- **Variant**: 템플릿에서 생성된 변형 문항. 원본 추적 가능
- **Assignment**: 학습지/과제. 목적/대상/문항 세트 포함
- **RecommendationEvent**: 추천 이벤트. 추천 타입 + 근거 + 교사 피드백

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 교사가 "목적에 맞는 문항 세트(5문항+유사문항)"를
  30초 이내에 완성할 수 있다.
- **SC-002**: 8만건 이상의 문항 데이터에서 검색 결과가
  1.5초 이내에 반환된다 (p95 기준).
- **SC-003**: 유사문항 검색에서 교사가 상위 5개 중 3개 이상을
  "관련 있음"으로 평가한다 (정밀도 60% 이상).
- **SC-004**: 자동 생성된 변형 문항의 CAS 자동 검증 통과율이
  95% 이상이다.
- **SC-005**: 자동 생성 문항 1건당 처리 시간이 10초 이내이다.
- **SC-006**: 모든 추천 결과에 설명 가능한 근거가 포함된다
  (근거 누락률 0%).
- **SC-007**: 교사가 처음 가입 후 5분 이내에 첫 문항 검색을
  완료할 수 있다 (온보딩 효율).
- **SC-008**: 시스템에 등록된 문항의 95% 이상에 스킬/성취기준/난이도
  메타데이터가 완전히 태깅된 상태를 유지한다.

## Clarifications

### Session 2026-04-07

- Q: RBAC에서 어떤 역할들이 필요한가? → A: 시스템 관리자 + 콘텐츠 검수자 + 교사(일반) 3역할. MVP는 관리자+검수자 중심, 교사 오픈은 이후 단계.
- Q: 학생이 시스템에 직접 접근하는가? → A: 아니오. 교사 전용 도구이며 학습지는 PDF/링크로 외부 배포. 학생 인증/응답 수집은 범위 밖.
- Q: 오개념 분류 체계의 출처는? → A: 연구 기반 사전 정의 목록으로 시작하고, 콘텐츠 검수자가 추가/수정 가능한 구조.
- Q: 문항 품질 상태 전이 규칙은? → A: 역전이 허용. draft→reviewed→approved→retired 순방향 + approved→draft 역전이 가능(수정 필요 시). 검수자가 draft↔reviewed↔approved 전이, 관리자가 retired 전이 및 역전이 승인.
- Q: MVP 난이도 체계 기준은? → A: 5단계 정수(1=매우 쉬움 ~ 5=매우 어려움), 검수자 수동 설정. IRT 파라미터는 학생 응답 데이터 축적 후 추가 보정.

## Assumptions

- 입력 문항의 수식은 이미 LaTeX 형식으로 추출/저작된 상태이다
  (HWP 파일에서의 직접 변환은 범위 밖).
- MVP 범위는 중학교 수학(대수/방정식)이며, 초등/고등은 이후 확장.
- 교사는 LaTeX 기본 문법을 이해하거나 수식 에디터를 사용할 수 있다.
- 한국 2022 개정 교육과정 데이터가 구조화 가능하다.
- 1-2명의 개발 팀으로 Phase 1부터 순차 구현한다.
- 시스템은 교사 전용 도구이다. 학생 직접 접근, 인증, 응답 수집 기능은 범위 밖이다.
- 수능/모의고사 기출 문항은 직접 저장하지 않고 유형/패턴 분석 후
  AI 유사문제 생성으로 대응한다 (저작권 준수).
- AI 생성물에 대해 2026 AI기본법 표시 의무를 준수한다.
